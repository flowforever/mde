# editor link 支持 - HIGH PRIORITY - READY FOR RELEASE

## 开发状态

* 2026-05-01: 已按 auto-pick-tasks 选中，开始分析 editor link 插入、内部/外部链接打开、UT/IT/E2E 覆盖和发布验证。
* 2026-05-01: 已实现 editor `/` Link 命令、路径建议、新建文档插入、内部/外部链接打开路由，并完成 lint、typecheck、unit、integration、E2E 验证。等待 v1.3.9 release 成功后归档。

## 完成说明

* `/` 菜单新增 Link 入口，弹窗支持按 `/` 分段搜索 Markdown 路径，上下键选择后回车或点击插入相对链接。
* Link 弹窗支持新建文档：默认选中当前文件目录，可从目录树选择目标目录，创建空 Markdown 文件后插入相对链接。
* 同工作区 Markdown 链接在当前窗口打开；已知其它工作区内的 Markdown 链接在新窗口打开对应工作区和文件；HTTP/HTTPS 链接通过系统浏览器打开。
* 新增安全边界：编辑器拒绝 `javascript:` href，外部链接 IPC 仅允许 HTTP/HTTPS。

## "/" 命令支持

* 用户敲下 "/" 的时候应该要有 link 选项

* 选择link 的时候应该要有输入框, 使用$huashu-design

  * 用户输入的时候应该类似 vscode (CMD + P) 路径智能提示一样, 按照 "/" 分割关键字 去列出匹配的路径

  * 列出的文档列表

    * 用户上下选择 选中之后按回车变成相对路径保存

  * 输入框旁边应该要有一个新建文档的按钮, $huashu-design

    * $huashu-design, 当用户选择新建文档的时候,显示出 跟左边panel 一样的 目录树菜单, 选中当前文件所在目录

    * 让用户双击选择目录再填写一个文件名, 回车先创建一个空的文档

## 内部link

* 文档链接属于同一个工作区, 应该在当前窗口打开

* 文档不属于同一个工作区,应该在新窗口打开

  * 如果文档有属于某个工作区应该打开工作区, 再打开对应文档

## 外部link

* 用户输入 http(s): 开头的链接,视为外部链接

* 在系统默认浏览器打开
